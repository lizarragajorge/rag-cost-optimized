P1

Cost-performance tuning

Tune chunking and text quality so you embed less noise and fewer duplicate tokens, while preserving retrieval quality.

Current vs Recommended

What they are doing now

Chunk split uses maximumPageLength 3500 and pageOverlapLength 600, with noisy OCR-like text appearing in chunks.

More cost-effective way

Reduce overlap, normalize text, and drop low-signal artifacts before embedding. Validate with a fixed relevance test set.

Why this works

Overlap and noise inflate token volume. Less duplicated and cleaner text lowers indexing cost while improving retriever signal.

How to implement

  1. Create a 30-50 query golden set with expected citations.
  2. Run baseline with current chunk settings and capture relevance metrics.
  3. Tune overlap downward in controlled steps and compare results.
  4. Add text cleanup filters in ingestion pipeline before chunking.
  5. Publish final settings and lock them in release config.

Tip: optimize for answer quality consistency, not just top-1 hit score.
