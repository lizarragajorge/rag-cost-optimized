P0

Immediate savings and risk control

Change the highest-cost behavior first: full refresh runs with expensive embeddings. This item alone usually moves the budget curve quickly.

Current vs Recommended

What they are doing now

Embedding skill uses text-embedding-3-large and broad refresh operations can reprocess large portions of the corpus.

More cost-effective way

Default to text-embedding-3-small and make incremental refresh the standard path; full rebuild only for schema/model migration events.

Why this works

Cost scales with embedding tokens. The fastest savings come from reducing model unit cost and reducing re-embedded volume.

How to implement

  1. Create a shadow index for A/B validation (small vs large model quality check).
  2. Update embedding deployment in skillset or pipeline to text-embedding-3-small.
  3. Move production refresh trigger to incremental endpoint only.
  4. Add approval policy for full rebuild runs.
  5. Publish weekly report showing run cost trend before/after change.

Tip: treat this as a release with rollback criteria based on answer quality and citation correctness.
