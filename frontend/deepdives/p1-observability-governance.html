P1

Observability and governance

Build run-level transparency so cost and reliability decisions are data-driven, not guesswork.

Current vs Recommended

What they are doing now

Costs are visible in aggregate billing, but run-level causality (which source/indexer caused spend) is limited.

More cost-effective way

Require telemetry per refresh run and per source profile, then alert on abnormal token or failure spikes.

Why this works

Governance requires attribution. Without per-run metrics, teams cannot optimize or enforce cost SLOs.

Recommended run payload

Response schema

{
  "runId": "...",
  "documentsSeen": 0,
  "documentsSkipped": 0,
  "chunksEmbedded": 0,
  "tokensEmbedded": 0,
  "costUsd": 0.0,
  "durationMs": 0,
  "failedItems": 0
}

Alert thresholds

- tokensEmbedded > 2x weekly median
- failedItems > 0
- durationMs > SLA threshold
- rerun count > daily limit

Ownership

- Product owner: quality gate
- Platform owner: run reliability
- FinOps owner: spend trends

How to implement

  1. Instrument refresh endpoint and indexer orchestration with structured logs.
  2. Push metrics to App Insights / Log Analytics and build dashboard by source.
  3. Define weekly review cadence for cost anomalies.
  4. Set policy gates for full rebuild execution and exception recording.

Tip: publish a one-page weekly scorecard to keep teams aligned on reliability + cost goals.
