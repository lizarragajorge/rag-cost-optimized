P3

Strategic optimization

After core savings and reliability are stable, optimize index architecture and enrichment footprint for long-term scale.

Current vs Recommended

What they are doing now

One broad index and enrichment profile handles mixed domains, source types, and volatility patterns.

More scalable way

Partition by domain where needed, tier refresh cadence by volatility, and remove unused enrichment skills.

Why this works

Separating workloads improves relevance quality, operational isolation, and long-term cost predictability.

Optimization map

Index segmentation

- regulatory
- operations
- legal/contracts

Use query-time routing by domain

Cadence tiers

- high volatility: daily
- medium: weekly
- archive: monthly/on-demand

Skill rationalization

- keep only fields used by:
  - retrieval filters
  - ranking
  - answer generation
- drop unused enrichments

How to implement

  1. Audit query logs to identify domain clusters and top filters.
  2. Propose target index topology and migration plan with no downtime.
  3. Run dual-write period and compare relevance/cost metrics.
  4. Retire unused skill outputs and update knowledge source mappings.
  5. Publish architecture decision record and ownership matrix.

Tip: prioritize operational simplicity over theoretical optimum if team bandwidth is constrained.
